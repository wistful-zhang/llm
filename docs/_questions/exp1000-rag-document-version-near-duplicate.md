## 面试时怎么答

说明内容去重不能抹掉版本语义，要维护版本链、生效区间和默认过滤。

**可以这样答：**

> 近重复检测可以发现版本关系，但不能简单只保留一份，因为历史查询可能需要旧版本。每份文档应记录逻辑文档 ID、版本号、生效与失效时间以及替代关系。默认检索只召回查询时点有效的最新版本，用户明确问历史时再切换时间过滤。生成上下文不能同时混入多个未标记版本，若条款发生变化应明确展示差异。

## 常见追问

1. **内容相似度多高算同一版本链？** 不能只设通用阈值，还要结合来源路径、标题、文档编号和人工规则。
2. **没有版本号怎么办？** 使用发布时间、内容哈希和来源事件建立内部版本，并标记推断的不确定性。
