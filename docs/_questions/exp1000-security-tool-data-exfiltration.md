## 面试时怎么答

先用自己的话说清“工具数据外传”，再给出一项可检查的证据或例子；结尾别漏掉加密或编码数据仍可能绕过简单关键词规则，需要控制能力而非只扫内容。

**可以这样答：**

> 对出站域名、参数字段和敏感数据分类实施允许列表与 DLP，默认禁止模型任意拼接外部请求。一个直接的检查办法是：在模拟工具中植入要求上传上下文的恶意指令并检查审计日志。这个结论的边界是，加密或编码数据仍可能绕过简单关键词规则，需要控制能力而非只扫内容。

## 常见追问

1. **如果只保留一个要点，“工具数据外传”是什么？** 对出站域名、参数字段和敏感数据分类实施允许列表与 DLP，默认禁止模型任意拼接外部请求
2. **给出一个可以复现或手工检查“工具数据外传”的办法。** 在模拟工具中植入要求上传上下文的恶意指令并检查审计日志
3. **在哪种条件下，“工具数据外传”会失效或被误读？** 加密或编码数据仍可能绕过简单关键词规则，需要控制能力而非只扫内容
