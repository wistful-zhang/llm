## 面试时怎么答

建议按“结论—例子—边界”回答“LRU Cache 实现”：核心判断要明确，验证要可复现，并说明并发版本需要锁或分片，缓存值的资源释放也要定义。

**可以这样答：**

> 这件事可以概括为：哈希表定位节点，双向链表维护最近使用顺序；访问或更新移到头部，超容量删除尾部。落到实验或实现上，测试更新已有键、容量一、重复访问和淘汰顺序。同时要确认，并发版本需要锁或分片，缓存值的资源释放也要定义。

## 常见追问

1. **“LRU Cache 实现”最需要讲清的核心内容是什么？** 哈希表定位节点，双向链表维护最近使用顺序；访问或更新移到头部，超容量删除尾部
2. **哪项具体检查可以支撑你对“LRU Cache 实现”的判断？** 测试更新已有键、容量一、重复访问和淘汰顺序
3. **“LRU Cache 实现”最容易被忽略的前提是什么？** 并发版本需要锁或分片，缓存值的资源释放也要定义
