## 面试时怎么答

建议按“结论 → 原理 → 取舍 → 落地”回答：

1. **先给结论**：先按 STAR 说业务规模、成本基线、质量护栏和自己的决策权，再报最终量化结果。
2. **再讲关键机制**：把成本拆输入、输出、模型、重试和 GPU 利用率，说明如何用 Pareto 实验选方案。
3. **主动说取舍**：承认路由误判、压缩丢证据、量化退化等风险，并讲回退与灰度而非只报收益。
4. **最后落到项目**：必须给每成功任务成本、质量区间、P99、回退率与月度节省，停下等追问细节。

> 使用提醒：下面只示范表达结构，不能照搬情节或数字。请替换成自己可解释、可核验的真实经历；没有亲历过就明确说“我没有直接负责过，下面给出我的优化方案”。

**60 秒口述示例：**

> 我会按真实事实这样组织：“【产品】原来每个成功任务成本是【基线】，我负责在【质量非劣界】下优化。拆账后最大的三项是【真实成本项】，我分别验证【方案】并保留低置信回退。离线成对评测通过后按【灰度比例】放量，最终每成功任务成本变为【结果】，质量区间、P99 和回退率是【真实数据】。我还补了【长期监控】。”没有真实项目时，应明确把它作为方案题回答。

## 核心回答

案例题的重点是证明优化可归因、质量约束真实且结果可复现。先给业务量、账单结构、原始质量与 SLO；再说明你如何将成本拆为模型单价或 GPU 时间、输入与输出 Token、工具调用、重试和空闲容量。按最大成本项提出假设，使用模型路由、缓存、Prompt/上下文压缩、输出控制、批处理、量化或专用小模型逐项消融。

“质量不降”应预先定义为主指标差值的置信区间不越过非劣界，同时安全、最差切片和延迟不过门禁。上线采用 Canary 和自动回退，结尾报告每成功任务成本而非每请求成本，并明确自己的贡献、一次失败尝试与长期监控。

## 展开说明

成本优化常见误区是缩短输出后每请求便宜，却导致用户追问增加，最终每会话更贵；或平均质量不变但长尾和高价值用户退化。路由器应评测误路由成本与覆盖率，缓存要考虑权限、新鲜度和隐私，量化要按任务与长度分桶回归。

## 工程实践

建立逐请求成本账本并聚合到会话和成功任务；先用流量回放估算上限，再分阶段上线。仪表盘同时展示质量差值区间、最差切片、回退率、Cache Hit、Token、P99 和月度成本。每个优化项保留 Feature Flag，出现护栏退化可单独关闭。

## 常见追问

1. **怎样证明质量真的没下降？** 在同题成对评测中预先设非劣界，报告差值置信区间，并确认安全与关键切片没有被平均值掩盖。
2. **为什么用每成功任务成本？** 便宜模型若导致更多重试和追问，每请求成本会给出假节省；成功任务成本更接近业务产出。
3. **模型路由失败时如何兜底？** 路由器输出校准置信度，低置信或高风险请求走强模型，并持续把误路由样本加入回归与再训练。

## 一句话复习

> 成本案例要先拆账、再做有质量护栏的单变量优化，以每成功任务成本、置信区间和回退机制证明收益。
