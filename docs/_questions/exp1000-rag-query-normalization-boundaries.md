## 面试时怎么答

说明纠错、格式统一的收益，同时强调专有名词、否定和时间条件必须保真。

**可以这样答：**

> 规范化可以统一大小写、全半角、日期格式和常见别名，减少索引形式差异造成的漏召回。用户的产品名、错误码、数字、否定词和限定时间不能被模型随意改写，否则会改变真实意图。较稳妥的做法是保留原 Query，并生成一个规范化版本作为额外召回通道。线上要分析改写前后召回差异，发现关键实体漂移时回退原文结果。

## 常见追问

1. **错别字一定要纠正吗？** 不一定，可同时用原词和候选纠错词检索，避免把合法的生僻实体改错。
2. **规范化应该在索引端还是查询端做？** 稳定规则最好两端一致，查询端还可针对当前意图做有限扩展。
