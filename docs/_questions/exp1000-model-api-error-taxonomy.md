## 面试时怎么答

建议按“结论—例子—边界”回答“模型调用错误分类”：核心判断要明确，验证要可复现，并说明不能把上游敏感响应直接透传，HTTP 状态与业务错误也需保持一致。

**可以这样答：**

> 区分客户端参数、鉴权、配额、内容策略、可重试上游故障、超时和内部 Bug，并返回稳定错误码与 request ID。要验证这一点，可以采用这个办法：为每类错误定义重试、降级和用户提示的契约测试。使用时不能忽略，不能把上游敏感响应直接透传，HTTP 状态与业务错误也需保持一致。

## 常见追问

1. **“模型调用错误分类”最需要讲清的核心内容是什么？** 区分客户端参数、鉴权、配额、内容策略、可重试上游故障、超时和内部 Bug，并返回稳定错误码与 request ID
2. **哪项具体检查可以支撑你对“模型调用错误分类”的判断？** 为每类错误定义重试、降级和用户提示的契约测试
3. **“模型调用错误分类”最容易被忽略的前提是什么？** 不能把上游敏感响应直接透传，HTTP 状态与业务错误也需保持一致
