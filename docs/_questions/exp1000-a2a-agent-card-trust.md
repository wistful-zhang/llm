## 面试时怎么答

区分发现元数据与安全认证，回答签名、来源白名单、运行时授权和结果验证。

**可以这样答：**

> Agent Card 是能力和端点描述，不等于该主体已通过组织信任与安全审核。客户端应从受信目录或经过验证的域获取，检查签名、证书和版本，并将声明能力映射到本地允许策略。委派时只发送完成子任务所需的最少数据与权限，返回结果仍按不可信外部输入验证。能力、身份或端点变化要触发重新审批，不能长期缓存后盲信。

## 常见追问

1. **HTTPS 域名可信就够了吗？** 不够，它证明连接到该域，不证明业务主体、能力质量或授权范围符合要求。
2. **外部 Agent 说任务完成了怎么验证？** 检查结构化结果、证据和可观察状态，不能只接受自然语言声明。
