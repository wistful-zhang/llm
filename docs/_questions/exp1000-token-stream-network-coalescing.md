## 面试时怎么答

解释小包、序列化和系统调用开销，并提出短时间或字符级合并与首包例外。

**可以这样答：**

> 逐 Token 发送会产生大量小包、序列化、锁和系统调用开销，客户端渲染也可能频繁抖动。可以让首个 Token 尽快送达，之后按几毫秒、若干 Token 或自然文本边界做小批合并。合并窗口过大又会恶化感知流畅度，因此要测 ITL 和前端显示效果。连接断开时应立即向推理调度器传播取消，避免模型继续生成无人消费的内容。

## 常见追问

1. **SSE 和 WebSocket 哪个更快？** 两者都可满足文本流，实际差异常小于代理、缓冲和实现，按交互方向与基础设施选择。
2. **代理把流缓存住怎么办？** 配置禁用响应缓冲并发送适当心跳，在端到端环境验证首包而非只测服务进程。
