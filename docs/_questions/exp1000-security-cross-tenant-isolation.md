## 面试时怎么答

组织答案时围绕“跨租户隔离”展开：把定义或机制讲清楚，用具体例子验证，并说明前端隐藏和 Prompt 声明都不是隔离，缓存与异步任务尤其容易漏边界。

**可以这样答：**

> 身份、缓存、向量检索、会话、日志和文件都必须携带不可伪造的租户边界，并在服务端强制。具体例子是，用两个测试租户交叉猜测 ID、缓存前缀和检索过滤器。真正落地前还要检查，前端隐藏和 Prompt 声明都不是隔离，缓存与异步任务尤其容易漏边界。

## 常见追问

1. **不铺背景，直接说明“跨租户隔离”的核心机制或判断。** 身份、缓存、向量检索、会话、日志和文件都必须携带不可伪造的租户边界，并在服务端强制
2. **把“跨租户隔离”落到一个可检查的例子，你会怎么做？** 用两个测试租户交叉猜测 ID、缓存前缀和检索过滤器
3. **什么情况下不能直接套用“跨租户隔离”？** 前端隐藏和 Prompt 声明都不是隔离，缓存与异步任务尤其容易漏边界
