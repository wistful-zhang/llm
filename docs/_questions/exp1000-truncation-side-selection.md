## 面试时怎么答

根据系统指令、近期对话和任务尾部问题等信息位置决定保留策略。

**可以这样答：**

> 右截断保留开头，适合系统规则和文档前部关键的任务，却可能删掉用户最后的问题。左截断保留最近内容，适合对话续写，但可能丢失系统指令和早期约束。生产系统应按消息结构和优先级裁剪，保护不可丢字段，并在截断后重新验证模板边界。

## 常见追问

1. **保留首尾、中间截断可行吗？** 可行，适合首部规则加尾部问题，但会破坏中间文档连续性。
2. **截断发生在字符层还是 Token 层？** 最终必须按 Token 预算控制，并避免切坏特殊模板或不可分割字段。
