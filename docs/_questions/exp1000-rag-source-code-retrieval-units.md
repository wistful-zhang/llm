## 面试时怎么答

比较不同粒度，并说明符号、调用关系和版本信息比纯文本相似度更重要。

**可以这样答：**

> 整文件保留上下文但常过长，函数或类通常是更合适的基本单元，语法树节点可进一步处理超大函数和跨语言结构。索引文本应包含符号名、签名、注释、路径和所属类，并建立定义、引用与调用图。查询先用符号和词法召回，再结合向量语义和依赖邻居扩展。所有结果必须绑定提交版本，避免把不同分支的代码拼在一起。

## 常见追问

1. **自动生成代码摘要再检索有用吗？** 可补充自然语言查询，但摘要可能遗漏细节，原始代码和符号索引仍要保留。
2. **第三方依赖要不要入库？** 按任务需要选择，通常保留版本化 API 文档和关键源码，避免无关依赖占满候选。
