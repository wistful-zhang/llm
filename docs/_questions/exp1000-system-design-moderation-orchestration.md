## 面试时怎么答

说明三阶段处理不同风险，并考虑流式安全缓冲、上下文和人工复核。

**可以这样答：**

> 输入审核识别禁止请求和高风险上下文，决定拒绝、限制工具或切换安全流程。生成中可对流式文本块增量检查，在展示前保留小缓冲，发现风险时停止后续生成。输出审核结合完整回答、用户场景和引用，处理单块无法判断的语义。策略结果结构化记录，边界案例转人工，并同时评估漏放与过度拦截。

## 常见追问

1. **审核模型和主模型可以相同吗？** 可以作基线，但会形成相关失败，关键场景更适合独立模型与规则组合。
2. **流式内容已经发出后发现违规怎么办？** 应尽量用安全缓冲提前发现，并支持停止、隐藏已发内容与事件处置。
