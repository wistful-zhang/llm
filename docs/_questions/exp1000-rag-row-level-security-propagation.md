## 面试时怎么答

回答权限属性随派生数据传播、查询时重新求值及缓存隔离。

**可以这样答：**

> 每个派生 Chunk 必须继承源行的租户、所有者、组织和策略版本，不能在聚合时丢掉权限。查询侧用可信身份计算允许范围，并把过滤条件下推到索引；跨多行生成的摘要只允许使用权限交集。权限模型变化时要重新标记或在线求值，旧缓存按主体和策略版本失效。审计日志记录谁在什么策略下命中了哪些文档，但避免保存不必要的原文。

## 常见追问

1. **多个权限不同的行能合成一个 Chunk 吗？** 不建议，除非明确采用最严格的权限交集，否则很容易发生越权。
2. **过滤条件太复杂导致向量召回慢怎么办？** 可按租户或安全域分区索引，并预计算稳定组权限，但不能为了性能取消授权。
