## 面试时怎么答

按事实类型定义来源权威性，不要设置一个覆盖所有场景的全局排序。

**可以这样答：**

> 可信优先级应针对具体字段定义，例如用户偏好以用户最新确认优先，账户权限以授权系统为准，产品规则以当前官方版本为准。Prompt 中为每条上下文附来源、时间和适用范围，让模型在冲突时按字段策略选择。不能把“数据库永远正确”写成全局规则，因为数据库也可能过期或不适合解释主观意图。高影响冲突应停止自动决策，展示差异并请求权威来源确认。

## 常见追问

1. **来源优先级应该由模型判断吗？** 模型可辅助解释，但正式优先级应由业务配置提供，避免随措辞变化。
2. **两份官方文档版本不同怎么办？** 比较生效时间、产品版本和地区范围，无法对应时标记冲突而不是任选一份。
