## 面试时怎么答

说明上传后先探测图像形状与切片数，再按总序列和 KV 预算准入。

**可以这样答：**

> 请求到达后先读取安全元数据并执行预处理规划，得到图片数、Tile 数和预计视觉 Token，而不是只按文件字节估算。调度器把视觉编码显存、投影后上下文长度和预计输出 KV Cache 一起纳入准入。实际 Token 超出估计时需要重新排队或降低分辨率，不能让整批 OOM。监控按图像复杂度分桶比较估计与真实用量，持续修正容量模型。

## 常见追问

1. **为什么文件大小不能预测视觉 Token？** 压缩率和内容不同，同样字节数可以对应完全不同的像素尺寸和切片数量。
2. **可以统一降到最小分辨率简化调度吗？** 会损害 OCR 和小目标任务，应提供质量档位而不是全局最低配置。
