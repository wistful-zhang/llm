## 面试时怎么答

回答“对冲请求”时，先给出核心结论，再补一个能检查结论的例子，最后主动说明这条边界：非幂等工具调用不能直接对冲，副本故障相关时收益有限；阈值过低还会形成正反馈，把已经过载的系统压得更慢。

**可以这样答：**

> 核心判断是，主请求超过基于历史分位数设置的延迟阈值后再向独立副本补发，采用首个有效结果并取消其余请求，可绕过偶发慢副本但会增加负载。实际验证可采用这个办法：在可控慢节点实验中比较无对冲、固定延迟对冲和自适应阈值下的 P99、额外 Token 与取消成功率。此外，非幂等工具调用不能直接对冲，副本故障相关时收益有限；阈值过低还会形成正反馈，把已经过载的系统压得更慢。
