## 面试时怎么答

覆盖选择、参数、顺序、冗余、恢复和副作用安全，说明分层指标便于定位。

**可以这样答：**

> 应分别统计工具选择准确率、参数合法率、首次调用成功率、平均调用次数和无效重复率。多步任务还要看依赖顺序、错误恢复、预算超限和是否产生未授权副作用。最终任务成功率是核心，但分层指标能说明失败来自模型、工具接口还是编排器。指标按工具、任务难度和风险等级切片，避免平均值掩盖少量严重事故。

## 常见追问

1. **调用次数越少越好吗？** 不是，必要验证不能省，目标是在保证正确和安全的前提下减少冗余。
2. **参数合法率高但任务失败说明什么？** 可能选错工具、计划顺序错误或工具数据不足，需要结合轨迹分析。
