## 面试时怎么答

说明先识别结构再设置软硬长度边界，并为标题和上下文补充元数据。

**可以这样答：**

> 可以先按标题、段落、列表和代码块识别自然边界，再用目标 Token 范围作为软约束。片段过长时在次级边界继续拆分，超过硬上限必须切开；片段过短则与相邻同主题内容合并。每个 Chunk 保留文档标题和层级路径，但不要机械复制大量祖先正文。切分效果要用真实问题的召回与答案完整性验证，而不是只看平均长度。

## 常见追问

1. **重叠窗口还需要吗？** 可对被硬切开的连续文本使用小重叠，结构化边界清楚时不必处处重叠。
2. **标题是否计入向量文本？** 通常应计入以补充语境，但需避免重复标题主导短 Chunk 的相似度。
