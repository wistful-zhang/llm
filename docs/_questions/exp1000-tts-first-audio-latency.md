## 面试时怎么答

拆解 LLM 首 Token、文本分句、声学模型和播放缓冲，说明流式切块的质量取舍。

**可以这样答：**

> 首音频延迟包含 LLM 生成、文本切分、TTS 首包和网络播放缓冲，必须逐段测量。可以让 LLM 流式输出，在稳定的短语或标点处尽早送入流式 TTS，同时预热模型和复用连接。切得太小会缺少韵律上下文并产生语调断裂，切得太大又增加等待。客户端使用有界抖动缓冲，并在用户打断时立即取消后续生成与播放。

## 常见追问

1. **收到几个字就立即合成好吗？** 通常不好，中文分词和语气尚未确定，容易造成停顿和重音异常。
2. **预生成固定开场白可行吗？** 可降低感知延迟，但不能误导用户，也要避免所有请求都播放无价值套话。
