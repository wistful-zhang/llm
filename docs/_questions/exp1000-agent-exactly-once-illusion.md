## 面试时怎么答

解释请求执行后响应丢失的二义性，给出至少一次投递加业务幂等。

**可以这样答：**

> 网络可能在服务端已执行后丢失响应，调用方无法仅凭超时判断是否发生副作用。消息系统和 Worker 故障也会导致重复投递，因此端到端 Exactly-once 通常无法只靠传输保证。工程上采用至少一次投递、稳定操作 ID 和工具端幂等记录，实现业务效果上的一次。无法幂等的动作需要状态查询、人工确认或把它放在受控的最后一步。

## 常见追问

1. **数据库唯一键能解决吗？** 能保护单库内创建重复，但跨服务副作用和外部发送仍需各自幂等协议。
2. **消息队列宣称 Exactly-once 呢？** 通常只覆盖特定队列处理语义，不自动覆盖外部工具的业务效果。
