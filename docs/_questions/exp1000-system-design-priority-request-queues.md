## 面试时怎么答

按交互 SLO 和可抢占性分池或加权调度，并防止低优先任务永久饥饿。

**可以这样答：**

> 交互请求需要低 TTFT，可使用保留容量或高权重队列；Agent 长任务允许步骤间暂停，离线批处理更适合填充剩余容量。调度按 Token 和显存预算而非请求数计费，过载时先延后可恢复任务。低优先队列采用老化与最低份额避免永久饥饿，高优先也有配额防止挤占全部资源。队列状态和预计等待时间向上层公开，便于用户取消或选择异步。

## 常见追问

1. **是否应该部署三套独立集群？** 隔离最清晰但利用率可能低，可按规模采用保留池加共享溢出容量。
2. **长任务被抢占后如何恢复？** 持久化步骤状态和输入，复用幂等键，按重算或 KV 迁移策略恢复。
