## 面试时怎么答

说明跨系统不能依赖数据库事务，要为每一步预定义补偿和不可逆边界。

**可以这样答：**

> 跨邮件、订单和日历系统通常无法使用一个 ACID 事务，应把流程建模为 Saga。每个正向动作记录结果和对应补偿，例如取消预订或撤回草稿，后续失败时按逆序执行。补偿本身也可能失败，需要幂等、重试和人工处理队列。不可逆步骤应尽量后置并在执行前确认，不能假设所有动作都能完全回滚。

## 常见追问

1. **退款算完全回滚付款吗？** 不一定，手续费、汇率和时间影响仍存在，应作为业务补偿而非真正事务回滚。
2. **补偿由模型临时想可以吗？** 不可以，高风险补偿应由工具作者预定义并经过测试，模型只负责选择允许路径。
