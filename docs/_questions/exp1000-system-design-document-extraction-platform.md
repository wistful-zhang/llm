## 面试时怎么答

描述安全上传、分类解析、字段校验、证据坐标和复核队列。

**可以这样答：**

> 上传层先做病毒、文件类型、大小和解压限制，再按文档类型选择 OCR、版面和 VLM 管线。抽取结果遵循版本化 schema，每个字段带原文页码、坐标和置信信息，金额日期等由规则做跨字段校验。低置信或高风险字段进入人工复核，修订同时保留原值与审核者。流水线幂等且按文档版本运行，模型或模板升级时可对受影响文档受控重算。

## 常见追问

1. **如何避免同一文档重复处理？** 用租户内内容哈希和业务文档 ID 去重，所有阶段使用确定性派生 ID。
2. **Schema 新增字段怎么办？** 发布新 schema 版本，旧结果保留，按业务需要增量回填而不是覆盖历史。
