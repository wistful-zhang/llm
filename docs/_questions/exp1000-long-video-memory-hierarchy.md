## 面试时怎么答

回答镜头级特征、事件摘要、时间索引和按需回看原片段。

**可以这样答：**

> 先按镜头和音频事件分段，为每段保存关键帧、转写、对象和时间范围。再构建章节级摘要与全局时间线，但所有摘要都保留指向原始片段的索引。用户提问时先检索相关事件，再按需加载更密集帧和音频进行二次判断。这样摘要提供导航而不充当唯一事实源，也能控制视觉 Token 成本。

## 常见追问

1. **镜头切分会漏掉跨镜头事件吗？** 会，因此上层事件需要允许覆盖多个镜头，并保留相邻窗口。
2. **摘要模型写错了怎么办？** 最终回答回看原始片段，并用时间戳引用，摘要只用于候选召回。
