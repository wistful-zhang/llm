## 面试时怎么答

覆盖事件顺序、幂等、事务边界、重放和最终一致性。

**可以这样答：**

> 每个源记录要有稳定 ID 和单调版本，索引消费者只接受比当前更新的事件。创建、修改和删除都转成幂等操作，重复投递不会产生重复 Chunk。跨表事务若影响同一文档，应等待一致快照或在聚合层重建，避免索引短暂拼出不存在的状态。消费者保存检查点和失败队列，支持从日志重放，并监控源版本与索引版本的滞后。

## 常见追问

1. **CDC 保证事件顺序就够了吗？** 还要考虑分区间顺序、消费者重试和索引写入失败，因此版本校验仍不可少。
2. **更新一行是否只重算一个向量？** 取决于切分和聚合关系，影响父文档语义时可能需要重建多个 Chunk。
