## 面试时怎么答

回答类型与范围校验、不可变版本、灰度、监控和原子回滚。

**可以这样答：**

> 配置进入版本库或配置平台前做类型、范围和跨字段校验，生产只读取审核后的不可变版本。发布先在测试流量验证，再按实例或租户灰度，观察质量、安全、延迟和成本指标。请求记录实际配置版本，方便复现，出现异常时原子切回旧版本。关键阈值设置系统安全上下限，操作者不能通过配置意外关闭硬防线。

## 常见追问

1. **热更新和重启加载哪个好？** 热更新快但要保证请求级一致性，复杂底层参数可通过滚动重启更安全。
2. **多个配置一起改怎么办？** 作为一个经过验证的配置包发布，避免实例看到半新半旧组合。
