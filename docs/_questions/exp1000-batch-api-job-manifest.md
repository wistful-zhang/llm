## 面试时怎么答

建议按“结论—例子—边界”回答“批任务清单”：核心判断要明确，验证要可复现，并说明只记录文件名不足以复现，输入内容哈希和代码版本同样重要。

**可以这样答：**

> 这件事可以概括为：清单应记录输入版本、模型配置、幂等 ID、输出位置、重试策略、权限和每条记录状态。落到实验或实现上，中断批任务后依据 manifest 只重跑未完成记录并核对条数。同时要确认，只记录文件名不足以复现，输入内容哈希和代码版本同样重要。

## 常见追问

1. **“批任务清单”最需要讲清的核心内容是什么？** 清单应记录输入版本、模型配置、幂等 ID、输出位置、重试策略、权限和每条记录状态
2. **哪项具体检查可以支撑你对“批任务清单”的判断？** 中断批任务后依据 manifest 只重跑未完成记录并核对条数
3. **“批任务清单”最容易被忽略的前提是什么？** 只记录文件名不足以复现，输入内容哈希和代码版本同样重要
