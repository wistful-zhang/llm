## 面试时怎么答

构建有状态模拟工具和可注入故障，按最终状态而不是文字判断成功。

**可以这样答：**

> 为工具实现确定性的有状态模拟器，支持创建、查询、冲突、权限错误和超时等真实语义。评测任务给定初始世界状态和目标状态，运行后由程序检查资源变化，而不是让模型评审对话是否好看。模拟器可注入乱序、重复响应和部分失败，测试恢复能力。它不能完全替代沙箱集成测试，因此少量真实环境用例仍要在隔离账号中验证。

## 常见追问

1. **模拟工具返回固定答案够吗？** 不够，无法测试多步状态变化、并发和错误恢复，应根据请求更新世界状态。
2. **怎样防止 Agent 记住模拟器规律？** 随机化非关键数据和场景组合，并保留隐藏任务集。
