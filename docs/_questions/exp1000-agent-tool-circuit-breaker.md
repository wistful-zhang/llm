## 面试时怎么答

解释 closed、open、half-open 状态，并让规划器看见能力暂不可用。

**可以这样答：**

> 持续失败的工具若被每个 Agent 反复调用，会放大下游故障并浪费模型预算。熔断器在失败率或延迟超过阈值时进入 open，快速返回明确的暂不可用错误，等待后用少量 half-open 探针恢复。工具注册表向规划器暴露当前健康状态，模型可选择安全替代或告知用户等待。熔断按依赖和错误类型细分，不能因单个无效参数关闭整个服务。

## 常见追问

1. **熔断和限流有什么区别？** 限流控制请求量，熔断在依赖异常时阻止继续调用，两者可以同时存在。
2. **谁决定替代工具是否等价？** 由预定义能力与数据范围配置决定，模型只能在允许候选中选择。
