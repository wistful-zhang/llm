## 面试时怎么答

先用自己的话说清“合并显存访问”，再给出一项可检查的证据或例子；结尾别漏掉缓存可能掩盖小测试差异，数据布局改变也可能增加别处开销。

**可以这样答：**

> 核心判断是，同一 Warp 线程访问连续且对齐地址时可合并成更少内存事务，跨步或随机访问会浪费带宽。实际验证可采用这个办法：比较行优先连续访问与按列跨步访问的带宽。此外，缓存可能掩盖小测试差异，数据布局改变也可能增加别处开销。

## 常见追问

1. **如果只保留一个要点，“合并显存访问”是什么？** 同一 Warp 线程访问连续且对齐地址时可合并成更少内存事务，跨步或随机访问会浪费带宽
2. **给出一个可以复现或手工检查“合并显存访问”的办法。** 比较行优先连续访问与按列跨步访问的带宽
3. **在哪种条件下，“合并显存访问”会失效或被误读？** 缓存可能掩盖小测试差异，数据布局改变也可能增加别处开销
