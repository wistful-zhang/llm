## 面试时怎么答

组织答案时围绕“模型响应适配层”展开：把定义或机制讲清楚，用具体例子验证，并说明把不同停止原因粗暴映射为成功会掩盖失败，适配层也必须随 SDK 版本回归。

**可以这样答：**

> 可以从这条主线理解：内部协议统一文本增量、工具调用、停止原因、用量和错误类型，同时保留供应商特有原始字段供诊断。再用一个最小验证说明：用同一契约回放正常流、并行工具、内容过滤和截断响应。最后明确限制：把不同停止原因粗暴映射为成功会掩盖失败，适配层也必须随 SDK 版本回归。

## 常见追问

1. **不铺背景，直接说明“模型响应适配层”的核心机制或判断。** 内部协议统一文本增量、工具调用、停止原因、用量和错误类型，同时保留供应商特有原始字段供诊断
2. **把“模型响应适配层”落到一个可检查的例子，你会怎么做？** 用同一契约回放正常流、并行工具、内容过滤和截断响应
3. **什么情况下不能直接套用“模型响应适配层”？** 把不同停止原因粗暴映射为成功会掩盖失败，适配层也必须随 SDK 版本回归
