## 面试时怎么答

说明由可信后端按单次动作签发能力，而不是把长期密钥放进模型上下文。

**可以这样答：**

> 模型永远不应看到长期 API 密钥，编排器在调用被批准后为具体工具、资源和动作签发短期凭证。凭证绑定用户、租户、最大范围和到期时间，服务端不能接受模型自行扩大的参数。敏感动作可使用一次性 capability，执行后立即失效。所有签发与使用事件进入审计，并支持快速撤销异常会话。

## 常见追问

1. **只把密钥放在环境变量里够吗？** 比放 Prompt 好，但若所有工具共享同一高权密钥仍不符合最小权限。
2. **工具服务器如何验证用户身份？** 验证由可信编排层签发的主体和范围声明，不信任模型生成的 user_id。
