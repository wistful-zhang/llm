## 面试时怎么答

不要给单一结论，要比较串行表达、并行效率、激活显存和稳定性。

**可以这样答：**

> 加深增加连续变换步数，可能更利于组合复杂计算，但串行依赖更长，训练稳定性和流水线延迟更难处理。加宽提高每层容量和矩阵乘规模，通常更容易吃满硬件，却会增加每个 Token 的激活和单层参数。最优形状取决于总算力、并行拓扑、数据规模和目标延迟，通常通过缩放实验而不是只按参数量决定。

## 常见追问

1. **为什么极窄极深模型不一定高效？** 小矩阵难以充分利用 GPU，层间串行启动和通信开销又会累积。
2. **推理延迟更敏感于深度还是宽度？** 单请求低批量时通常更敏感于层数，因为每层形成连续关键路径。
