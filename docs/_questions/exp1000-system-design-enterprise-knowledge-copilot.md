## 面试时怎么答

先明确权限、时效和引用要求，再按入库、检索、生成、评测与反馈拆服务。

**可以这样答：**

> 离线侧负责连接数据源、解析、版本化、权限传播和多路索引，在线侧负责身份认证、查询路由、检索重排、生成与引用校验。每次查询在检索前执行用户权限过滤，模型只接收获准证据，答案句子绑定可访问来源。缓存按租户、权限和索引版本隔离，知识更新驱动失效。系统还要有离线评测、线上反馈、审计和无答案降级，不能把全部可靠性压在一个 Prompt 上。

## 常见追问

1. **知识更新后多久可见？** 按来源定义新鲜度 SLA，并监控从源变更、解析、索引到查询命中的各段延迟。
2. **员工离职后如何处理权限？** 身份系统撤权后检索立即拒绝，相关主体缓存和会话上下文同步失效。
