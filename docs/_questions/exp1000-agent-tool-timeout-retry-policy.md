## 面试时怎么答

区分连接前失败、响应丢失和服务端已执行，副作用工具优先用幂等查询。

**可以这样答：**

> 超时只说明客户端没及时拿到结果，不代表服务端没有执行。只读且幂等的请求可有限重试，有副作用的调用必须复用幂等键，或先用操作 ID 查询状态。重试采用指数退避和随机抖动，并受总截止时间约束。若状态未知且无法安全确认，应停止自动重试并提示用户，不能冒险重复付款或发送。

## 常见追问

1. **HTTP 500 都可以重试吗？** 不一定，要看错误码是否标记可重试以及操作幂等性，参数错误重试无益。
2. **客户端断线后怎么办？** 服务端继续按幂等操作记录状态，客户端重连后用操作 ID 查询。
