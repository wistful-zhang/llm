## 面试时怎么答

使用乐观并发版本和幂等操作回答，并说明冲突时不能静默最后写入获胜。

**可以这样答：**

> 每份会话状态携带 revision，提交下一步时要求客户端提供它读取到的版本。服务端以 compare-and-swap 原子更新，版本不一致就拒绝并返回最新状态。副作用调用使用独立操作 ID，即使两个请求竞争也不会重复执行。冲突发生后让用户选择合并或重新执行，不能用最后写入覆盖已确认结果。

## 常见追问

1. **数据库事务能完全解决吗？** 能保护单次写入，但外部工具和用户交互跨度长，仍需版本与幂等设计。
2. **只允许一个页面连接可以吗？** 可降低复杂度，但断线和后台任务仍会并发，服务端不能省略保护。
