## 面试时怎么答

覆盖规范化 Query、索引版本、过滤条件和主体权限，不要只按用户问题缓存。

**可以这样答：**

> 缓存键至少包含规范化 Query、检索配置、索引版本、租户和会影响结果的权限过滤摘要。只按自然语言问题缓存会把不同用户可见结果混在一起，也会在索引更新后返回旧文档。高频稳定查询可使用短 TTL，并在重要文档或权限变更时主动失效。缓存内容仍按敏感数据保护，命中后再次验证当前主体是否有权访问每条结果。

## 常见追问

1. **能跨用户共享公共文档的缓存吗？** 可以，但必须证明数据和过滤策略完全公共，并使用独立公共命名空间。
2. **语义相似 Query 能共享缓存吗？** 风险较高，因为细微条件可能改变答案，需严格校准并保留实体与过滤一致性检查。
