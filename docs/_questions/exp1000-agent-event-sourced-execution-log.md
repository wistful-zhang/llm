## 面试时怎么答

说明追加事件可重建状态和审计，但要处理敏感数据、schema 演进和副作用不可重放。

**可以这样答：**

> 事件溯源把用户输入、计划决策、工具请求、结果和状态转移作为不可变事件追加，便于审计与重建执行状态。它能区分当时发生了什么和当前派生状态，也支持离线分析失败路径。代价是事件 schema 需要版本化，敏感内容要最小化与加密，长期存储成本也更高。重放只能重建状态，不能再次执行真实副作用，工具调用必须用记录结果或模拟器替代。

## 常见追问

1. **事件可以修改以满足删除请求吗？** 可用加密擦除或去标识化处理敏感载荷，同时保留必要的非敏感审计骨架。
2. **快照有什么作用？** 定期保存派生状态，避免每次都从最早事件重放，事件仍作为事实记录保留。
