## 面试时怎么答

说明多维预算由编排器强制执行，并在规划时提供剩余额度和降级策略。

**可以这样答：**

> 请求开始时设定 Token、工具调用、费用、墙钟时间和并行度上限，编排器在每步前原子扣减或预留。模型可以看见剩余预算以调整计划，但不能自行提高额度。接近上限时优先完成核心目标、减少探索或返回部分结果，不能突然静默中断。预算耗尽、用户取消和系统超时都进入明确终止状态，并清理正在运行的工具。

## 常见追问

1. **工具费用执行后才知道怎么办？** 调用前按最坏或预计值预留，完成后结算差额，超限动作需确认。
2. **高价值任务能自动加预算吗？** 只能在预先授权范围内调整，超过租户配额必须由可信策略或用户批准。
