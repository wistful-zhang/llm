## 面试时怎么答

回答按视觉形状和 Token 长度分桶、分别编码与动态拼批，并关注尾延迟。

**可以这样答：**

> 直接把差异很大的请求放进同一批次会产生大量图像与序列 Padding，浪费显存和算力。调度器可按图片数量、分辨率档位和预计总 Token 分桶，视觉编码阶段与语言解码阶段也可分开批处理。等待更匹配批次能提高吞吐，但会增加排队时间，因此需要最大等待窗口。性能评测要用真实尺寸分布，报告吞吐和 P99，而不是只测统一方图。

## 常见追问

1. **视觉塔输出后还能重新拼批吗？** 可以，缓存每个请求的视觉特征，再按语言序列预算组合，但要管理生命周期和显存。
2. **多图请求是否应该拆成多次视觉编码？** 可以并行编码，但最终仍需保留图片顺序和请求归属。
