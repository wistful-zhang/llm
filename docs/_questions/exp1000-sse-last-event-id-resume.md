## 面试时怎么答

建议按“结论—例子—边界”回答“SSE 可恢复流”：核心判断要明确，验证要可复现，并说明模型生成本身未必可从任意位置恢复，事件续传与重新推理要区分。

**可以这样答：**

> 服务端为事件分配单调 ID 并短暂保留，客户端重连携带最后确认 ID，服务端从下一条续传或明确要求重启。例如，在任意 Token 边界断网并检查拼接结果是否重、漏或乱码。需要注意的是，模型生成本身未必可从任意位置恢复，事件续传与重新推理要区分。

## 常见追问

1. **“SSE 可恢复流”最需要讲清的核心内容是什么？** 服务端为事件分配单调 ID 并短暂保留，客户端重连携带最后确认 ID，服务端从下一条续传或明确要求重启
2. **哪项具体检查可以支撑你对“SSE 可恢复流”的判断？** 在任意 Token 边界断网并检查拼接结果是否重、漏或乱码
3. **“SSE 可恢复流”最容易被忽略的前提是什么？** 模型生成本身未必可从任意位置恢复，事件续传与重新推理要区分
