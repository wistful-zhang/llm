## 面试时怎么答

建议按“结论 → 原理 → 取舍 → 落地”回答：

1. **先给结论：** SVD 把矩阵分解为正交方向与奇异值，保留前 r 个最大奇异值可得到 Frobenius/谱范数下的最佳 rank-r 近似。
2. **再讲关键机制：** 写出 `W=UΣVᵀ` 与截断 `W_r=U_rΣ_rV_rᵀ`，说明参数从 `mn` 变为约 `r(m+n)`。
3. **主动说取舍：** 权重低秩不等于功能误差小，截断节省也可能被两个小矩阵的 kernel 开销抵消。
4. **最后落到项目：** 按层扫描 rank，报告重构误差、任务掉点、显存、吞吐和 P95 时延；说完停。

**60 秒口述示例：**

> 我会先写 `W=UΣVᵀ`：U、V 给出左右正交方向，Σ 的奇异值按重要性排序。截断到前 r 项得到 `W_r`，Eckart–Young 定理保证它在 Frobenius 或谱范数下是最佳 rank-r 近似，参数量约从 `mn` 降到 `r(m+n)`。但权重误差不等于任务误差。项目里我会逐层选 rank，并测验证掉点、显存、tokens/s 和 P95。 并检查两个小矩阵在目标 GPU 上是否真正高效执行。

## 核心回答

对 `W∈R^{m×n}`，SVD 为 `W=UΣVᵀ`，奇异值 `σ_1≥σ_2≥...≥0`。保留前 r 个奇异值和对应向量，得到 `W_r=U_rΣ_rV_rᵀ`。Eckart–Young–Mirsky 定理说明，它是所有 rank 不超过 r 的矩阵中，在谱范数和 Frobenius 范数下逼近 W 最好的一个。

低秩形式可写成两个矩阵乘法，例如 `A=U_rΣ_r`、`B=V_rᵀ`，参数从 `mn` 降为 `r(m+n)`。只有 rank 足够小时参数数目才真正下降；实际速度还依赖矩阵形状、批量和硬件 kernel。

## 展开说明

Frobenius 重构误差满足 `||W-W_r||_F²=Σ_{i>r}σ_i²`，谱范数误差为 `σ_{r+1}`，因此奇异值衰减能直接给出线性代数层面的可压缩性。不过神经网络有层间补偿和非线性，一层权重的最小重构误差不保证端到端 loss 最小。

LoRA 不是直接对原权重做 SVD 截断，而是假设任务更新 `ΔW` 可用低秩矩阵学习；低秩压缩则通常近似已有 W。两者动机相近但优化对象和使用阶段不同。

## 工程实践

先画每层奇异值能量累计曲线，再用校准数据测激活加权误差，避免只按权重矩阵统一 rank。对 Q/K/V、FFN 和 Embedding 分别做敏感性实验。部署时确认两个小 GEMM 是否高效；必须报告真实显存、吞吐和尾延迟。

## 常见追问

1. **特征值分解与 SVD 有什么区别？** 特征值分解通常要求方阵且不一定有正交基；SVD 适用于任意矩阵，奇异值非负。
2. **为什么截断 SVD 是最佳低秩近似？** 正交奇异方向按能量排序，丢弃尾部造成的最小 Frobenius 误差正是尾部奇异值平方和。
3. **低秩参数更少为什么可能不更快？** 计算被拆成两个 kernel，启动、中间读写和不友好的小矩阵形状可能抵消 FLOPs 节省。
4. **SVD 压缩与 LoRA 是一回事吗？** 不是。前者分解已有权重做压缩，后者冻结基座并训练低秩增量用于适配。

## 一句话复习

> SVD 用奇异值排序矩阵方向，截断前 r 项给出线性代数意义上的最佳 rank-r 近似。
