## 面试时怎么答

列出目标、输入边界、输出 schema、权限、预算和完成条件，并要求结果可验证。

**可以这样答：**

> 契约应写清单一目标、可用输入、禁止访问的数据、允许工具、预算、截止时间和输出 schema。还要定义成功、部分成功和失败状态，以及返回证据或产物的位置。子 Agent 不能自行扩大任务或再委派未授权能力，需求不清时返回问题。主 Agent 验证结果满足契约后才合并，不能因文本自信就视为完成。

## 常见追问

1. **需要把全部上下文都传给子 Agent 吗？** 不需要，只传完成任务所需信息，减少隐私、噪声和注入传播。
2. **子任务失败后谁决定重试？** 协调器根据错误类型和预算决定，子 Agent 可建议但不能无限自重试。
