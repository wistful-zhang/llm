## 面试时怎么答

从工具边界、候选缩减和调用前检查回答，不要把选择完全交给工具名称。

**可以这样答：**

> 工具定义应写清适用场景、不适用场景、数据范围和副作用，让模型能看到真正差异。工具很多时先按意图和权限筛出少量候选，再将完整 schema 提供给模型。调用前由代码检查请求对象、资源类型和用户授权是否与工具匹配。持续统计工具混淆矩阵，把真实误选案例补进描述和回归集。

## 常见追问

1. **直接把两个工具合成一个可以吗？** 只有参数和权限边界一致时才适合，否则大而模糊的工具会更难安全调用。
2. **能让模型先解释为什么选这个工具吗？** 可输出简短选择理由用于诊断，但仍需程序校验，理由不是授权依据。
