## 面试时怎么答

说明键由一次业务意图生成并跨重试复用，服务端保存结果而不是只做短时间去重。

**可以这样答：**

> 幂等键应对应一次明确业务动作，在首次决定执行时生成，网络超时后的所有重试继续使用同一个键。服务端把键与主体、操作类型、规范化参数和最终结果绑定，参数不同却复用同一键时应拒绝。键的保存时间覆盖可能重试窗口，关键交易通常要更久。Agent 收到已处理响应时复用原结果，不能再次创建副作用。

## 常见追问

1. **用请求时间戳当幂等键可以吗？** 不适合，重试会产生新时间戳，无法识别同一业务意图。
2. **读取工具需要幂等键吗？** 通常不需要，但触发昂贵异步任务或有隐藏副作用的读取接口仍要考虑。
