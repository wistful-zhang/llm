## 面试时怎么答

说明状态机约束合法转移、持久化进度和副作用，聊天文本只作输入。

**可以这样答：**

> 聊天记录无法可靠表达当前步骤、已确认动作和哪些副作用已经执行。显式状态机定义合法状态与转移条件，使模型不能跳过审批或重复提交。每次转移原子持久化输入、结果和版本，重启后可从确定位置恢复。自然语言负责理解用户意图，真正推进流程仍由状态机验证。

## 常见追问

1. **开放式 Agent 也能用状态机吗？** 可用高层通用状态，如规划、等待工具、等待确认和完成，内部计划仍可动态。
2. **状态很多会不会难维护？** 可以用层级状态和子工作流拆分，仍比隐含在 Prompt 中更可审计。
