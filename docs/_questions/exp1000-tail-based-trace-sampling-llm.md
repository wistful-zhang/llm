## 面试时怎么答

这题不要只报术语。先解释“高流量 Trace 采样”的核心逻辑，再说明如何验证，最后指出尾部采样需要暂存 Span 且不能突破隐私边界，原始 Prompt 不应因为调试而默认长期保留。

**可以这样答：**

> 可以从这条主线理解：入口可做低比例随机采样，链路结束后再按错误、慢请求、特定版本和罕见路径提高保留率，并记录采样概率用于统计校正。再用一个最小验证说明：把全量短期基线与采样结果比较，检查错误类型、延迟分位数和租户分布是否仍可还原。最后明确限制：尾部采样需要暂存 Span 且不能突破隐私边界，原始 Prompt 不应因为调试而默认长期保留。
