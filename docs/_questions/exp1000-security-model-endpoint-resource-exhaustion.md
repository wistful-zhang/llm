## 面试时怎么答

这题不要只报术语。先解释“模型端点资源耗尽防护”的核心逻辑，再说明如何验证，最后指出只限制请求次数挡不住单请求放大，公开上限也需防止攻击者探测内部容量。

**可以这样答：**

> 在入口限制字节、Token、并发、输出和工具轮次，按身份计费与限流，并让排队和取消能及时释放 GPU 资源。一个直接的检查办法是：生成长输入、慢客户端、最大输出和循环工具组合，观察队列与费用上限。这个结论的边界是，只限制请求次数挡不住单请求放大，公开上限也需防止攻击者探测内部容量。
