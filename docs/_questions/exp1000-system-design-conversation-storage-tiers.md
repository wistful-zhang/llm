## 面试时怎么答

区分事实源和派生数据，回答版本、血缘、权限与删除传播。

**可以这样答：**

> 原始消息是受权限保护的事实源，结构化记忆和摘要都是带来源与版本的派生数据，向量只用于检索。热消息放低延迟存储，旧记录可归档，但读取与删除语义保持一致。每个派生项记录输入消息 ID 和生成版本，源内容修改或删除时可精确失效重建。保留期限按数据类型和用户授权设置，不能因为向量不可读就长期保留。

## 常见追问

1. **摘要能替代归档原文吗？** 取决于产品和合规需求，摘要有损且可能错误，不能默认作为法律或审计记录。
2. **不同设备如何同步？** 服务端以 revision 排序和合并，客户端离线写入发生冲突时显式处理。
