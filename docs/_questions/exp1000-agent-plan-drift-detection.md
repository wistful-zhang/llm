## 面试时怎么答

要求保存不可变目标和约束，在关键步骤比较当前状态与完成条件。

**可以这样答：**

> 任务开始时把用户目标、禁止事项和验收条件抽取为独立契约，后续计划都引用该契约而不是递归摘要。每次高成本或有副作用动作前，检查它是否直接推进某个未完成条件。工具结果导致目标变化时必须由用户或可信策略确认，不能让外部内容偷偷改写任务。周期性生成进度表，发现连续步骤无贡献或出现额外目标就重规划。

## 常见追问

1. **目标抽取错了怎么办？** 在执行前向用户展示关键目标与范围，高风险任务要求确认。
2. **用户中途改变需求呢？** 建立新契约版本，明确哪些旧步骤保留、取消或需要补偿。
