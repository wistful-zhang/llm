## 面试时怎么答

建议按“结论—例子—边界”回答“批内负例”：核心判断要明确，验证要可复现，并说明Batch 偏小、跨设备未聚合或假负例过多都会降低效果。

**可以这样答：**

> 核心判断是，一个 Batch 中其他样本可复用为负例，大幅减少编码成本并增加候选数。实际验证可采用这个办法：在多正例数据中用样本 ID 掩掉同义或重复文本。此外，Batch 偏小、跨设备未聚合或假负例过多都会降低效果。

## 常见追问

1. **“批内负例”最需要讲清的核心内容是什么？** 一个 Batch 中其他样本可复用为负例，大幅减少编码成本并增加候选数
2. **哪项具体检查可以支撑你对“批内负例”的判断？** 在多正例数据中用样本 ID 掩掉同义或重复文本
3. **“批内负例”最容易被忽略的前提是什么？** Batch 偏小、跨设备未聚合或假负例过多都会降低效果
