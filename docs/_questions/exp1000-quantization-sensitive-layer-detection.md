## 面试时怎么答

说明逐层替换、输出误差和任务指标，不要只按层号经验保留。

**可以这样答：**

> 可以一次只量化某层或某类模块，比较层输出误差、困惑度变化和下游任务退化。输入输出 Embedding、少数离群通道和靠近输出的层有时更敏感，但不能当成固定规律。根据敏感度把少量模块保留在更高位宽，其余使用低比特，再评估整体速度和显存。混合精度若导致频繁格式转换，质量提升可能被 kernel 开销抵消。

## 常见追问

1. **只看权重重建误差够吗？** 不够，小重建误差也可能影响关键路径，应结合激活和任务输出。
2. **每层都测试成本太高怎么办？** 先按模块类型和统计特征筛选候选，再对高风险层做精细消融。
