## 面试时怎么答

组织答案时围绕“工具调用评测”展开：把定义或机制讲清楚，用具体例子验证，并说明语法正确不代表业务安全，副作用操作还需权限与确认评测。

**可以这样答：**

> 可以从这条主线理解：分开判断是否该调用、工具是否选对、参数是否合法、值是否来自上下文以及结果是否被正确使用。再用一个最小验证说明：用模拟工具记录调用轨迹并注入缺参、错参和无需调用样本。最后明确限制：语法正确不代表业务安全，副作用操作还需权限与确认评测。

## 常见追问

1. **不铺背景，直接说明“工具调用评测”的核心机制或判断。** 分开判断是否该调用、工具是否选对、参数是否合法、值是否来自上下文以及结果是否被正确使用
2. **把“工具调用评测”落到一个可检查的例子，你会怎么做？** 用模拟工具记录调用轨迹并注入缺参、错参和无需调用样本
3. **什么情况下不能直接套用“工具调用评测”？** 语法正确不代表业务安全，副作用操作还需权限与确认评测
