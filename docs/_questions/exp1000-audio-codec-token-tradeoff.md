## 面试时怎么答

说明时间帧率、码本数量和残差量化共同决定 Token 数与重建质量。

**可以这样答：**

> Codec 通常按固定时间帧把音频编码为一个或多个离散码本，帧率和码本数量越高，重建细节通常越好但序列更长。多码本残差量化能逐级补充信息，却增加并行或自回归建模复杂度。低码率可能损失音色、辅音和背景声，高码率则显著提高训练与推理成本。选择应比较感知质量、说话人相似度、可懂度和实时率，而不只看波形误差。

## 常见追问

1. **Codec Token 和文本 Token 能共用词表吗？** 可以放在统一序列中，但通常保留不同 ID 区间和模态嵌入，避免语义混淆。
2. **为什么语音序列特别长？** 音频按时间密集采样，即使低帧率也远多于一句话的文本 Token 数。
