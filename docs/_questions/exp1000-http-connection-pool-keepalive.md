## 面试时怎么答

口述“HTTP 连接复用”时，先回答它是什么或怎么做，再用例子落地，最后交代适用限制：连接复用不当会遇到陈旧连接、DNS 更新和资源未关闭，流式响应也会长期占槽。

**可以这样答：**

> 这件事可以概括为：复用 Keep-Alive 连接可减少 TCP 与 TLS 握手，但连接池必须限制总数、每主机并发、空闲时间和获取等待。落到实验或实现上，比较每请求新建客户端与共享客户端的握手次数、P95 和端口使用。同时要确认，连接复用不当会遇到陈旧连接、DNS 更新和资源未关闭，流式响应也会长期占槽。

## 常见追问

1. **请用自己的话说明“HTTP 连接复用”的核心做法。** 复用 Keep-Alive 连接可减少 TCP 与 TLS 握手，但连接池必须限制总数、每主机并发、空闲时间和获取等待
2. **你准备怎样举例证明自己理解“HTTP 连接复用”？** 比较每请求新建客户端与共享客户端的握手次数、P95 和端口使用
3. **使用“HTTP 连接复用”前还要确认什么？** 连接复用不当会遇到陈旧连接、DNS 更新和资源未关闭，流式响应也会长期占槽
