## 面试时怎么答

先绑定精确请求版本与上下文，再做去隐私、反作弊、归因和人工治理。

**可以这样答：**

> 每条反馈关联请求、模型、Prompt、检索证据和展示版本，避免把点踩归因到错误组件。自由文本和用户改写先做授权、PII 处理和恶意内容隔离，不能直接进入训练。分析时校正只愿意反馈的用户偏差，并按检索、事实、风格和安全失败分类。经过抽样审核的数据进入评测或训练版本，发布后用独立集合验证是否真正改善。

## 常见追问

1. **点踩就代表答案错误吗？** 不一定，可能是不喜欢风格、延迟或产品交互，需要结合原因和行为信号。
2. **用户改写能直接当标准答案吗？** 不能，可能包含新事实、隐私或错误，应经过来源与质量审核。
