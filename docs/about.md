# 关于这份笔记

这是一个可复用的大模型面试准备与复盘网站。你可以浏览题库、随机模拟、直接在网站 UI 中增加题目、在浏览器记录真实面试进度，也可以使用同一套项目建立属于自己的公开或私有题库。

## 整理原则

- **直接回答问题**：开头就回应题目，不用先介绍自己准备怎么答。
- **按题型展开**：原理题讲因果，设计题讲约束，项目题讲真实证据。
- **只说真实经历**：没有做过的项目可以讲推演，但不要编造指标和事故。
- **准备追问**：把面试官可能继续深挖的方向列出来。
- **练习口述**：60 秒是上限，不是必须填满；说清楚后就收尾，再对照参考说法复盘。

## 来源与核验

内置题目有两种证据状态：232 道“资料核验”题已检查来源和技术依据；768 道扩展题用于补足不同方向的覆盖面，虽然已有参考回答，但尚未逐题核对真实面经频率和一手资料。因此它们会显示“参考答案待校对”，不会被包装成某家公司原题或统一高频题。

带“资料核验”标记的题目已经检查题目来源、七段答题结构、追问接法和参考链接，具体依据列在每道题末尾。核验表示内容有可追溯资料，不表示不同团队会采用完全相同的标准答案；面试时仍应结合岗位和项目经验说明取舍。

1000 道题用于覆盖不同岗位和知识边界，不表示 1000 道都同等重要或都应默认学习。本站另用“核心必会 / 岗位专项 / 扩展知识点 / 待重整”标记建议备考顺序；它与题目难度、答案核验、是否公开和个人复习状态相互独立。四级含义、证据边界和不同岗位的学习路径见[学习分级说明]({{ '/study-tiers/' | relative_url }})。

## 如何使用

第一次准备时，先按[学习分级]({{ '/study-tiers/' | relative_url }})筛选“核心必会”，再进入[模拟面试]({{ '/practice/' | relative_url }})按目标岗位和分类抽题：先口头回答，再查看参考答案并标记掌握程度；也可以浏览单题，点击“开始口述”，之后用“面试时怎么答”和“常见追问”检查表达与知识盲点。

临时想到问题时，可以打开[增加题目]({{ '/capture/' | relative_url }})并直接选择谁能看。“只留给自己”只保存到当前浏览器；“公开给大家”会在隐私检查后先去 GitHub 查重，确认没有相同题目后，再从题目卡打开预填表单并最终提交。两种方式都允许先只写问题，答案以后再补。

想补充另一种答法、继续追问或纠错时，直接打开对应题目，在参考答案下方使用“评论与补充”。每一道题使用自己的评论线程，不再另设社区页面。题目上的“参考答案待校对”表示已经可读、可练，只是尚未逐题对照一手资料，并不是题目还在等待发布。

真实面试后，可以打开[面试记录]({{ '/interviews/' | relative_url }})保存公司匿名代号、岗位、轮次、结果、下一步和个人复盘。记录默认只在当前浏览器中可见，不会写入 GitHub；换设备前需要导出 JSON 备份。每个流程都可以保持“仅当前浏览器”，也可以标记为“准备匿名分享”；后者仍然不会自动上传，必须先生成匿名预览，再通过[公开面经发布步骤]({{ '/experiences/manage/' | relative_url }})二次确认。已经明确公开的内容会显示在[公开面经]({{ '/experiences/' | relative_url }})。

如果想建立自己的题库，请查看[使用指南]({{ '/start/' | relative_url }})。整个项目不依赖固定用户名或仓库名。

本站如何使用浏览器存储和外部服务，见[隐私说明]({{ '/privacy/' | relative_url }})。
