# 建立自己的大模型面经

这个项目是一个 GitHub 模板。每个人都可以创建独立仓库，使用自己的 GitHub 账号管理内容，不需要复制原作者的用户名或网站地址。

<div class="choice-grid">
  <div class="choice-card" markdown="1">
  ### 公开题库

  适合分享、求职复习和社区协作。仓库与网页都对互联网公开，可使用免费的 GitHub Pages。
  </div>
  <div class="choice-card" markdown="1">
  ### 私人题库

  适合个人笔记或敏感内容。创建私有仓库并且不要启用公开 Pages，只通过 GitHub 与 Pages CMS 管理。

## 创建步骤

1. 在当前项目的 GitHub 仓库页面点击 **Use this template → Create a new repository**。
2. 选择自己的 GitHub 账号、仓库名以及 Public 或 Private。
3. 若要发布公开网站，打开 **Settings → Pages**，将 Source 设为 **GitHub Actions**。
4. 打开[题库管理后台]({{ '/manage/' | relative_url }})，用 GitHub 登录并且只授权刚创建的仓库。
5. 在“站点设置”中修改名称和介绍，然后在“面试题与解答”中添加内容。

## 为什么使用模板而不是 Fork

模板会创建一个历史独立的新仓库，并允许你自行选择公开或私有。公开仓库的 Fork 永远是公开的，不能改为私有，因此 Fork 更适合向原项目提交改进，不适合保存个人私密笔记。

## 隐私提醒

私有仓库不等于私有网页。除 GitHub Enterprise Cloud 的私有 Pages 访问控制外，发布到 GitHub Pages 的网站通常应视为公开内容。不要把公司机密、个人信息、账号凭据或受保密约束的面试原题发布到 Pages。
