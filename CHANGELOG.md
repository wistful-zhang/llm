# 更新记录

这个文件记录会影响模板副本、数据格式或首次配置流程的变化。个人副本不会自动获得这些更新，应用前请先阅读 [UPGRADING.md](UPGRADING.md)。

## 1.5.0 - 2026-07-22

- 承认千题只代表覆盖面：新增“核心必会 / 岗位专项 / 扩展知识点 / 待重整”四级学习顺序，与难度、答案核验和发布状态分开。
- 通用核心清单收紧为 100 道，首页和随机模拟默认只使用这一层；岗位专项、扩展和待重整题仍可手动查看。
- 新增学习分级说明页，公开分级依据、证据边界、新手路径和不同岗位的选题方法。
- 768 道扩展题的来源说明改为“尚无逐题真实面经频率证明”，不再让覆盖题看起来像同等高频原题。
- Pages CMS 中可直接选择备考层级；新题默认进入待重整，旧副本缺少字段时仍可正常构建。

## 1.4.0 - 2026-07-22

- 取消独立“社区”产品入口：公开协作回到题目本身，旧 `/community/` 地址只保留迁移说明。
- 每道正式题目下方增加独立评论区，以稳定题目 slug 对应唯一 GitHub Issue；页面直接展示公开评论，并从 GitHub 完成登录和留言。
- “＋题目”表单新增“只留给自己 / 公开给大家”选择，默认只保存在当前浏览器；公开前必须确认隐私，且仍需在 GitHub 做最终提交。
- 正式题库列表上方新增“使用者新增的题目”，显示主动公开的问题、作者、更新时间和评论数，并提供全部公开补充入口；答案允许留空。
- 本机题目数据向后兼容新增 `visibility` 字段，旧记录一律迁移为 `private`，不会因升级意外公开。

## 1.3.0 - 2026-07-22

- 新增“面经社区”：访客用中文 Issue Form 投稿题目或参与校对，不需要 Markdown；每条帖子都可继续评论。
- 社区页通过当前仓库的公开 Issues API 展示最新投稿、作者、更新时间和评论数；不会收集 Token，私有仓库、限流或网络失败时保留 GitHub 回退入口。
- 题目页为未逐题核验的答案提供直接校对入口，并将容易误解的“答案待复核”统一改为“参考答案待校对”；题目与答案本身仍然正常发布和练习。
- 明确社区帖子立即公开、正式题库单独收录，以及 Public / Private 仓库下投稿和评论的隐私边界。

## 1.2.0 - 2026-07-22

- 题库扩展到 1000 道：保留 232 道资料已核验题，新增 768 道带口述回答与追问的参考答案待校对题。
- 补强数学、编程与框架、训练排障、推理部署、评测安全以及项目与行为面，不再只堆概念比较题。
- 首页增加难度和答案核验状态筛选，每次显示 60 道，并使用浏览器延迟渲染降低千题列表的绘制开销。
- 增加可复现的批次生成、总量检查、精确去重和近似标题审计工具；批量答案不会冒充逐题资料核验。

## 1.1.0 - 2026-07-21

- 新增站内“＋记题”：可用普通表单新增、编辑、搜索和练习题目，答案允许留空，不要求 Markdown。
- 题目草稿按仓库保存在当前浏览器，支持 JSON 备份与恢复、跨标签页冲突保护以及损坏数据救援。
- 支持从面试流程显式提取被问题目；不会带入公司、岗位、复盘等个人求职信息。
- 提供安全 Markdown、Codex 补答与发布指令、GitHub Issue 投稿入口；本页不收集 GitHub 密码、PAT 或 Token，也不会自动上传。

## 1.0.0 - 2026-07-18

- 提供 232 道已核验的大模型面试题、全文搜索、公式渲染和随机模拟面试。
- 提供浏览器本地面试流程、公司与轮次统计、JSON 恢复和 CSV 导出。
- 增加每轮被问题目记录、批量整理入口和匿名公开面经二次确认流程。
- 增加 Pages CMS 中文表单、手动检查发布按钮和 Public / Private 新手图解。
- 增加公开内容隐私与脚本注入校验、构建产物扫描和浏览器 CSP。
- 明确模板、Fork、浏览器本地存储、第三方 CMS 与独立副本升级边界。
