# 参与贡献

感谢帮助完善大模型面经。这个仓库同时包含网站模板和示例题库，请根据目标选择合适的方式。

## 建立自己的题库

请使用仓库页面上的 **Use this template**，不要通过 Pull Request 把个人笔记提交到公开题库。模板创建的仓库完全独立，可以选择公开或私有。

## 向公开题库贡献

可以通过两种方式：

1. 使用 GitHub Issue 表单提交题目和参考答案，维护者整理后收录。
2. Fork 仓库、增加或修改 `docs/_questions/` 中的内容，然后提交 Pull Request。

## 内容标准

一份高质量题目建议包含：

- 明确的“结论 → 原理 → 取舍 → 落地”回答顺序，以及可直接口述的 60 秒示例
- 准确但不要求逐字背诵的核心回答
- 原理、流程或公式的展开说明
- 工程实践中的选择、指标与取舍
- 至少 3 个常见追问，并在每个问题后给出 1～2 句直接接法
- 一句话复习结论

请优先使用自己的表达并核对事实。不要提交：

- 公司内部资料、未公开题库或受保密协议约束的内容
- 个人信息、账号凭据、访问 Token 或其他秘密
- 未获授权的大段版权内容
- 与大模型面试无关的推广信息

整理公开面经时，请只提炼高频主题并重新组织答案，在正文末尾列出可访问的公开来源与主要技术依据。匿名社区帖子适合说明“有人被问到过”，不能单独证明它是某公司的固定真题；不要把汇总题库标成未经核实的公司原题。

维护者完成题目来源、技术事实、七段答题结构、追问接法和参考链接检查后，可以加上 `verified: true`。它会在阅读网站显示“资料核验”标记，并触发更严格的自动校验；普通速记和个人题目保持省略或 `false` 即可。

## 文件格式

复制 `docs/_templates/question.md`，文件名使用简短、稳定的英文或数字 slug。推荐优先使用以下常见分类，也可以增加含义明确的新分类：

- LLM 基础
- 训练与对齐
- RAG
- Agent
- 工程实践
- 系统设计

提交到当前公开题库的题目应使用正式分类，难度必须是 `简单`、`中等` 或 `困难`，并设置 `review_status: 待复习`（或 `已掌握`）与 `published: true`。`待整理`、`待评估` 和空答案只用于个人仓库中的未发布速记。

## 提交前检查

- 页面标题、分类、难度、复习状态、标签和日期完整
- 内部链接有效，代码块指定合适语言
- 没有敏感信息和无权公开的内容
- 一个 Pull Request 尽量只处理一个主题
